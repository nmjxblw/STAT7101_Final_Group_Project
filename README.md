密码学：
置换密码：26字母随机打乱（26! 可能性）。可以使用频率分析及条件频率分析推断置换规则（26字母出现频率不同，E最多Z最少，且取决于之前的字母，这里为了简化可以假设一个单词内字母具有马尔可夫性）。

cryptor_and_dataset.ipynb：加密器与数据集生成

text.txt：训练文本

数据集：
每一行对应crypt类的一个随机种子（第一行对应seed 0），种子生成随机的密钥(key)和对应的逆密钥(inv_key)，从text中随机截取8000字符并用这个密钥加密，统计密文的字母频率。即每行的明文略有不同、每行的密钥完全不同。

频率：
[0, 25]：a-z每个字母在密文中的占比
[26, 51]：已知前一个字母为a，后一个字母是a-z的占比
[52, 77]：已知前一个字母为b，后一个字母是a-z的占比
以此类推。

目标：使用 [freq_0, freq_701] 预测 inv_key。
单点准确率定义：len([ i for i in range(26) if inv_key[i] == pred[i] ]) / 26
允许预测中出现重复字母，如 [‘a’ for _ in range(26)] 准确率为1/26
